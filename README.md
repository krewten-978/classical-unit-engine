# CUE: Classical Unit Engine

CUE is a repeatable workflow for building serious classical English, literature, philosophy, and history units without turning them into worksheets or shallow summaries.

It is designed for a teacher who wants students to read meaningful texts, compare real arguments, form judgments, and produce written or visual work: papers, window notes, dialogues, summaries, seminar claims, and argument maps.

CUE is not a generic lesson-plan generator. It is a disciplined production system for moving from a topic or text to a classroom-ready unit.

## Core promise

Serious intellectual material faster, without making it shallow.

## What CUE produces

A complete CUE pass can produce:

1. Unit brief
2. Source discovery packet
3. Recommended reading sequence
4. Source cards with citations
5. Student-facing reading handouts
6. Vocabulary support
7. Argument maps
8. Comparison maps
9. Paper prompts
10. Differentiated 10th/12th grade versions
11. After-teaching notes for future revision

## What CUE does not produce

CUE avoids:

- generic worksheets
- fill-in-the-blank busywork
- shallow summaries in place of readings
- teacher scripts masquerading as curriculum
- overstuffed units with too many weak sources
- modernized readings that distort the author’s real argument
- “activities” that do not result in student thought, writing, drawing, or argument

## How to use CUE conversationally

Use phrases like:

- “Run this through CUE.”
- “CUE this topic, but stop after source discovery.”
- “Make a 10th grade handout from this source.”
- “Build the argument map before we write the paper prompt.”
- “Do not build the unit yet; just prepare the source packet.”

## Built test unit

The first built CUE test is `units/do-we-love-shadows/`.

Use it to inspect:

- how a unit folder is organized
- how teacher-facing and student-facing materials are separated
- how the five-reading ladder becomes student packets
- how end exercises vary by text while still producing serious student work
