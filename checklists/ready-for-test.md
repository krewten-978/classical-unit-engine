# CUE Ready-for-Test Checklist

Before running the first real test topic, the repository should contain:

- [x] core principles
- [x] end-to-end workflow
- [x] source discovery standard
- [x] reading conversion standard
- [x] argument mapping standard
- [x] paper prompt design standard
- [x] differentiation standard
- [x] folder conventions
- [x] unit folder templates
- [x] source card template
- [x] handout plan template
- [x] argument map template
- [x] paper prompt template
- [x] intake prompt
- [x] run-CUE prompt

## First test should verify

- [ ] Can CUE turn a topic into a strong central question?
- [ ] Can CUE find at least six serious source candidates?
- [ ] Are source cards useful enough for Wayne to judge?
- [ ] Does CUE avoid overstuffing the unit?
- [ ] Does CUE choose conversion modes based on real difficulty?
- [ ] Are student-facing materials free of teacher notes?
- [ ] Do tasks produce student work rather than worksheet answers?
- [ ] Does the argument map reveal real disagreement?
- [ ] Does the paper prompt require evidence and counterargument?
- [ ] Are 10th/12th versions differentiated without dumbing down?

## Suggested first-test command

```text
Run CUE on [topic]. Stop after source discovery and recommended reading sequence. Do not build the full unit yet.
```
